## Overview

PulsarPay is a programmable payment layer built on **Solana** that enables seamless, pay-per-request interactions between users and autonomous agents. By combining deep crypto infrastructure (On-ramp, Exchange, and Yield) with an agile API, we allow agents to monetize services without the friction of traditional subscriptions.

## The Flow: From Setup to Monetization (happy path)



1.  **Setup:** User deposits funds (or buys crypto) and creates an API Key scoped to a specific agent.
2.  **Request:** User triggers a service (e.g., image generation).
3.  **Authorization & Charge Creation:** The agent attempts to create a charge via PulsarPay. `POST /charges`.
PulsarPay internally:
* Validates the API Key.
* Ensures it is scoped to the requesting agent.
* Checks if the user has sufficient balance
4.  **Execution:** Once the charge is authorized, the agent performs the requested service and returns the result to the user (e.g., generates and delivers the image).
5.  **Monetization:** Agent collects earnings through `POST /withdraw`.

## Security First

We prioritize the safety of your funds and the reliability of your agents.

* **Scoped API Keys:** Assign keys to specific agents to isolate spending.
* **Real-time Validation:** Every request is backed by a live balance check.
* **Controlled Access Model:** Users retain control over which agents can access their funds through explicit authorization.
