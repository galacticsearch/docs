Pulsarpay connects users and integrations through a real-time payment layer, enabling services to charge per request and deliver value instantly.

## How it works 

The Pulsarpay ecosystem relies on two main actors:
1. **Users:** Users fund their balance, generate API keys, and authorize providers to charge their account for each request.
2. **Providers:** Providers integrate with Pulsarpay to charge users in real time when delivering services. Charged amounts are credited to their balance and can be withdrawn on demand..

## Authentication & Authorization

Authentication requirements depend on the type of request.
Pulsarpay uses a dual-key model for operations that involve charging a user:

`x-agent-key`
Identifies the authorized integration (agent, service, or API) performing the request.

`x-user-key`
Represents the user who has authorized the integration to charge their balance.


## Charge Requests

For endpoints that perform charges (e.g. /agents/charges), both headers are required.

A charge will only be processed if:

1. The agent key is valid
2. The user key is valid
3. The user has sufficient balance
4. The agent is authorized to charge on behalf of the user

## How It Works

The user key acts as a delegated authorization, allowing the agent to charge the user's balance each time a request is executed.

### This model enables:

* Secure, per-request billing
* Clear separation between who executes and who pays
* Seamless integration of third-party services and agents
